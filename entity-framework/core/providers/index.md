# <a name="database-providers"></a><span data-ttu-id="0b413-102">資料庫提供者</span><span class="sxs-lookup"><span data-stu-id="0b413-102">Database Providers</span></span>

<span data-ttu-id="0b413-103">Entity Framework Core 能夠透過稱為資料庫提供者的外掛程式程式庫存取各種不同的資料庫。</span><span class="sxs-lookup"><span data-stu-id="0b413-103">Entity Framework Core can access many different databases through plug-in libraries called database providers.</span></span>

## <a name="current-providers"></a><span data-ttu-id="0b413-104">目前的提供者</span><span class="sxs-lookup"><span data-stu-id="0b413-104">Current providers</span></span>

> [!IMPORTANT]  
> <span data-ttu-id="0b413-105">EF Core 提供者是透過各種來源所建置。</span><span class="sxs-lookup"><span data-stu-id="0b413-105">EF Core providers are built by a variety of sources.</span></span> <span data-ttu-id="0b413-106">[Entity Framework Core 專案](https://github.com/aspnet/EntityFrameworkCore)的維護並不包含所有提供者。</span><span class="sxs-lookup"><span data-stu-id="0b413-106">Not all providers are maintained as part of the [Entity Framework Core Project](https://github.com/aspnet/EntityFrameworkCore).</span></span> <span data-ttu-id="0b413-107">考慮使用提供者時，請務必評估品質、授權、支援等項目，以確保其符合您的需求。</span><span class="sxs-lookup"><span data-stu-id="0b413-107">When considering a provider, be sure to evaluate quality, licensing, support, etc. to ensure they meet your requirements.</span></span> <span data-ttu-id="0b413-108">同時也務必檢閱每個提供者的文件以了解詳細的版本相容性資訊。</span><span class="sxs-lookup"><span data-stu-id="0b413-108">Also make sure you review each provider's documentation for detailed version compatibility information.</span></span>

> [!IMPORTANT]  
> <span data-ttu-id="0b413-109">EF Core 提供者通常可跨次要版本運作，但無法跨主要版本運作。</span><span class="sxs-lookup"><span data-stu-id="0b413-109">EF Core providers typically work across minor versions, but not across major versions.</span></span> <span data-ttu-id="0b413-110">例如，針對 EF Core 2.1 發行的提供者應該可用於 EF Core 2.2，但無法用於 EF Core 3.0。</span><span class="sxs-lookup"><span data-stu-id="0b413-110">For example, a provider released for EF Core 2.1 should work with EF Core 2.2, but will not work with EF Core 3.0.</span></span>
