## <a name="introduction-to-collations"></a><span data-ttu-id="2b286-109">定序簡介</span><span class="sxs-lookup"><span data-stu-id="2b286-109">Introduction to collations</span></span>

<span data-ttu-id="2b286-110">文字處理中的基本概念是定*序，這*是一組規則，可決定文字值的排序方式，並比較是否相等。</span><span class="sxs-lookup"><span data-stu-id="2b286-110">A fundamental concept in text processing is the *collation*, which is a set of rules determining how text values are ordered and compared for equality.</span></span> <span data-ttu-id="2b286-111">例如，雖然不區分大小寫的定序會略過大寫和小寫字母之間的差異，但對等比較而言，區分大小寫的定序則不會。</span><span class="sxs-lookup"><span data-stu-id="2b286-111">For example, while a case-insensitive collation disregards differences between upper- and lower-case letters for the purposes of equality comparison, a case-sensitive collation does not.</span></span> <span data-ttu-id="2b286-112">不過，由於區分大小寫區分文化特性（例如 `i` ， `I` 在土耳其文中以不同的字母表示），因此有多個不區分大小寫的定序，每一個都有自己的一組規則。</span><span class="sxs-lookup"><span data-stu-id="2b286-112">However, since case-sensitivity is culture-sensitive (e.g. `i` and `I` represent different letter in Turkish), there exist multiple case-insensitive collations, each with its own set of rules.</span></span> <span data-ttu-id="2b286-113">定序的範圍也會延伸至字元資料的其他層面，但不區分大小寫。例如，在德文中，有時候（但不一定）會想要將 `ä` 和視為 `ae` 相同。</span><span class="sxs-lookup"><span data-stu-id="2b286-113">The scope of collations also extends beyond case-sensitivity, to other aspects of character data; in German, for example, it is sometimes (but not always) desirable to treat `ä` and `ae` as identical.</span></span> <span data-ttu-id="2b286-114">最後，定序也會定義文字值的*排序*方式：雖然德文 `ä` 的地方 `a` ，瑞典文會將它放在字母的結尾。</span><span class="sxs-lookup"><span data-stu-id="2b286-114">Finally, collations also define how text values are *ordered*: while German places `ä` after `a`, Swedish places it at the end of the alphabet.</span></span>

<span data-ttu-id="2b286-115">資料庫中的所有文字作業會使用定序（不論是明確或隱含）來判斷作業比較和排序字串的方式。</span><span class="sxs-lookup"><span data-stu-id="2b286-115">All text operations in a database use a collation - whether explicitly or implicitly - to determine how the operation compares and orders strings.</span></span> <span data-ttu-id="2b286-116">可用定序和其命名配置的實際清單是資料庫專屬的;請參閱[下面的一節](#database-specific-information)，以取得各種資料庫相關檔頁面的連結。</span><span class="sxs-lookup"><span data-stu-id="2b286-116">The actual list of available collations and their naming schemes is database-specific; consult [the section below](#database-specific-information) for links to relevant documentation pages of various databases.</span></span> <span data-ttu-id="2b286-117">幸運的是，資料庫通常允許在資料庫或資料行層級定義預設定序，以及明確指定哪些定序應用於查詢中的特定作業。</span><span class="sxs-lookup"><span data-stu-id="2b286-117">Fortunately, database do generally allow a default collation to be defined at the database or column level, and to explicitly specify which collation should be use for specific operations in a query.</span></span>
